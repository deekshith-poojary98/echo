Introduction

Echo is a statically-typed, Python-based programming language with a clean and expressive syntax. It draws inspiration from C-style structure, modern scripting languages, and Pythonic readability. Designed to be beginner-friendly yet powerful, Echo supports strong type annotations, readable control flow, and simplified syntax for rapid development. 

fn greet(name: str) -> void {
    say("Hello, ", name);
}

greet("Echo");

x: int = 10;
y: float = 20.5;
name: str = "Alice";
is_valid: bool = true;

if x > 5 {
    say("x is greater than 5");
} else {
    say("x is 5 or less");
}

for i in 0..5 {
    say("Loop value:", i);
}

This example demonstrates Echo's basic syntax including function declarations, conditionals, loops, and type-safe variables.

Variables

Variables in Echo are strongly typed and must be declared with their type before use. The syntax follows the pattern variable_name: type = value.

Variable Declaration

Here's a comprehensive example of declaring different types of variables:

// Basic types
x: int = 10;            // Integer variable
y: float = 100.12;      // Float variable
name: str = "John";    // String variable
is_ready: bool = true;    // Boolean variable
anything: dynamic = 10;   // Dynamic variable

// Collections
numbers: list = [1, 2, 3];  // List of integers
mixed: list = [1, "two", 3.0];  // List with mixed types
user: hash = {
    "name": "Alice",
    "age": 25,
    "active": true
};  // Hash (HashMap) with string keys

Variable Reassignment

Variables can be reassigned, but the new value must match the declared type:

x = 20;  // Valid: same type (int)
name = "Bob";  // Valid: same type (str)
// x = "hello";  // Error: cannot assign string to int

// Dynamic variables can change type
anything = "Now I'm a string!";
anything = 42;
anything = true;
Important Notes
  • All variables must be declared with a type
  • Variables cannot change their type after declaration (except dynamic)
  • Variables have block scope and are not accessible outside their block (WIP)
  • Variable names must start with a letter or underscore and can contain letters, numbers, and underscores
  • Variable names are case-sensitive
Best Practices
  • Use descriptive variable names that indicate their purpose
  • Declare variables as close as possible to where they are used
  • Use dynamic type sparingly and only when necessary
  • Initialize variables when declaring them when possible
  • Use constants for values that shouldn't change

Data Types

Echo is a statically-typed language that supports several built-in data types. Each type has specific characteristics and use cases.

int

Integer numbers for whole-value arithmetic operations and counting.

  • Range: -2^63 to 2^63-1
  • No decimal points
  • Supports all arithmetic operations
age: int = 25;
count: int = -100;
result: int = age + count;
float

Floating-point numbers for precise decimal calculations.

  • 64-bit double precision
  • Supports scientific notation
  • Handles decimal arithmetic
pi: float = 3.14159;
scientific: float = 1.23e-4;
result: float = pi * 2.0;
str

String data for text manipulation and storage.

  • UTF-8 encoded
  • Immutable
  • Supports string interpolation
name: str = "Alice";
message: str = "Hello, " + name;
multiline: str = "Line 1\nLine 2";
bool

Boolean values for logical operations.

  • Values: true or false
  • Used in conditionals
  • Result of comparisons
is_valid: bool = true;
is_empty: bool = false;
result: bool = is_valid && !is_empty;
list

Ordered collections of elements.

  • Zero-based indexing
  • Mutable
  • Can contain mixed types
numbers: list = [1, 2, 3];
mixed: list = [1, "two", 3.0];
hash

Key-value mappings for structured data.

  • String keys only
  • Mutable
  • Supports nested structures
user: hash = {
    "name": "Alice",
    "age": 25
};
user["age"] = 26;  // Update value
dynamic

Flexible type for runtime type changes.

  • Can hold any type
  • Type checked at runtime
  • Use with caution
value: dynamic = 10;
value = "hello";
value = true;
Type Safety
  • All variables must be explicitly typed
  • Type checking is performed at compile time
  • Type mismatches result in compilation errors
  • Use dynamic only when necessary
  • Type annotations are mandatory for function parameters and return types

Type Conversion

Echo provides built-in functions for type conversion:

  • asInt(): Convert to integer
  • asFloat(): Convert to float
  • asString(): Convert to string
  • asBool(): Convert to boolean
// String to integer conversion
num_str: str = "123";
num: int = num_str.asInt();

// Float to string conversion
price: float = 19.99;
price_str: str = price.asString();

// String to float conversion
temp_str: str = "98.6";
temperature: float = temp_str.asFloat();

// Integer to boolean conversion
value: int = 1;
flag: bool = value.asBool();  // true

// String to boolean conversion
str_true: str = "true";
bool_value: bool = str_true.asBool();

Control Flow

Echo provides comprehensive control flow structures to manage program execution. These include conditional statements, loops, and control transfer statements.

Conditional Statements

Use if, else if, and else statements to control program flow based on conditions. All conditions must evaluate to a boolean value.

if user_age > 18 {
    say("User is an adult.");
} else if user_age == 18 {
    say("User is exactly 18 years old.");
} else {
    say("User is a minor.");
}

// Nested conditionals
if is_logged_in {
    if has_permission {
        say("Access granted.");
    } else {
        say("Insufficient permissions.");
    }
} else {
    say("Please log in first.");
}

While Loop

The while loop executes a block of code repeatedly as long as a condition is true. Use with caution to avoid infinite loops.

// Basic while loop
count: int = 0;
while count < 5 {
    say("Count:", count);
    count = count + 1;
}

// While loop with break
found: bool = false;
index: int = 0;
while !found && index < items.length() {
    if items[index] == target {
        found = true;
        break;
    }
    index = index + 1;
}

For Loop

The for loop iterates through a range of values with optional step increment using the by keyword. Use .. for inclusive ranges (includes end value) and ... for exclusive ranges (excludes end value). The break statement exits the loop early, while continue skips to the next iteration. Loop variables must be explicitly typed. 

// Basic for loop (exclusive range)
for i: int in 0...10 {
    say("Iteration:", i);  // Prints 0 to 9
}

// For loop with step (inclusive range)
for i: int in 0..10 by 2 {
    say("Even iteration:", i);  // Prints 0,2,4,6,8,10
}

// Reverse for loop (inclusive range)
for i: int in 10..0 by -1 {
    say("Countdown:", i);  // Prints 10 down to 0
}

// Loop with break statement
for i: int in 1...100 {
    if i > 5 {
        break;  // Exit loop when i > 5
    }
    say("Number:", i);  // Prints 1 to 5
}

// Loop with continue statement
for i: int in 1...5 {
    if i % 2 == 0 {
        continue;  // Skip even numbers
    }
    say("Odd number:", i);  // Prints 1, 3
}

Foreach Loop

The foreach loop iterates through all elements in a collection such as a list or hash. Loop variables must be explicitly typed. 

// Foreach with list
fruits: list = ["apple", "banana", "cherry"];
foreach fruit: str in fruits {
    say("Fruit:", fruit);
}

// Foreach with string
text: str = "Hello, World!";
foreach char: str in text {
    say("Character:", char);
}

// Foreach with hash
user: hash = {
    "name": "Alice",
    "age": 25,
    "city": "New York"
};
foreach key: str in user {
    say(key, ":", user[key]);
}
Control Flow Tips
  • Always use curly braces {} for control structures
  • Be careful with infinite loops in while loops
  • Use break and continue sparingly
  • Consider using foreach instead of indexed loops when possible
  • Keep loop bodies small and focused
Best Practices
  • Use meaningful variable names in loops
  • Keep conditions simple and readable
  • Use early returns in functions to reduce nesting
  • Consider using guard clauses for error conditions
  • Document complex control flow with comments

Functions

Functions in Echo are defined using the fn keyword, with mandatory type annotations for parameters and return types. The syntax is:

fn function_name(param1: type1, param2: type2) -> return_type { ... }

Return Types

Every function must specify a return type using the -> operator. Echo supports the following return types:

  • void - For functions that don't return a value
  • int - For functions returning integer values
  • float - For functions returning floating-point numbers
  • str - For functions returning strings
  • bool - For functions returning boolean values
  • list - For functions returning lists
  • hash - For functions returning hash maps
  • dynamic - For functions that may return different types
Important Notes
  • All functions have strict type checking for their parameters
  • Type conversion functions will raise errors if conversion is not possible
  • String functions only work on string values and will raise TypeError otherwise
  • Collection functions have specific type requirements (e.g., length() works on strings, lists, and hashes)
  • The wait() function is useful for creating delays in animations or timed operations

Type Safety

Echo enforces strict type checking for function returns. If a function declares a specific return type, it must return a value of that type, or a TypeError will be raised. Here are some examples: 

// Function with void return type - doesn't return a value
fn greet(name: str) -> void {
    say("Hello, ", name);
}

// Function that returns an integer
fn calculate_sum(a: int, b: int) -> int {
    return a + b;  // Returns int as specified
}

// Function that returns a string
fn get_full_name(first: str, last: str) -> str {
    return first + " " + last;
}

// Function with dynamic return type
fn get_value(key: str) -> dynamic {
    if key == "age" {
        return 25;  // Returns int
    } else {
        return "unknown";  // Returns string
    }
}

Type Errors

The following examples demonstrate common type errors that Echo will catch:

// Error: Function declares int return type but returns float
fn calculate_average(a: int, b: int) -> int {
    return (a + b) / 2.0;  // TypeError: Expected int, got float
}

// Error: Function declares void return type but returns a value
fn print_message(msg: str) -> void {
    say(msg);
    return "done";  // TypeError: void function cannot return a value
}

// Error: Missing return type annotation
fn add_numbers(a: int, b: int) {  // SyntaxError: Return type annotation required
    return a + b;
}
Best Practices
  • Always specify return types for clarity and type safety
  • Use void for functions that perform actions but don't return values
  • Use dynamic sparingly, only when a function truly needs to return different types
  • Match the declared return type exactly - type coercion is not automatic
  • Consider breaking down complex functions that might return different types into smaller, type-specific functions
Important Notes
  • The void type can only be used as a return type, not as a variable type
  • All functions must have an explicit return type annotation
  • Type checking is performed at runtime to ensure type safety
  • Return type mismatches will raise a TypeError

I/O Functions

Echo provides built-in functions for input/output operations and program control.

Input/Output Functions
  • ask(prompt: str) -> str: Prompts the user for input with the given message and returns the user's input as a string.
  • say(...values: dynamic) -> void: Prints values to the console, accepting multiple arguments of any type.
  • wait(seconds: float) -> void: Pauses execution for the specified number of seconds.
// Input and output examples
name = ask("What's your name? ");
say("Hello, ", name);
wait(1.5);  // Wait for 1.5 seconds

String Functions

Echo provides a set of built-in functions for string manipulation and processing.

String Manipulation
  • trim(text: str) -> str: Removes whitespace from both ends of a string.
  • upperCase(text: str) -> str: Converts all characters in a string to uppercase.
  • lowerCase(text: str) -> str: Converts all characters in a string to lowercase.
  • length() -> int: Returns the length of a string.
  • reverse() -> str: Reverses the characters in a string.
// String manipulation examples
text: str = "  Hello World  ";
clean: str = trim(text);  // "Hello World"
upper: str = upperCase(clean);  // "HELLO WORLD"
lower: str = lowerCase(clean);  // "hello world"
reversed: str = reverse(clean);  // "dlroW olleH"

Collection Functions

Echo provides comprehensive functions for working with lists and hashes.

List Methods

  • push(value: dynamic) -> list: Adds an element to the end of the list.
  • empty() -> list: Removes all elements from the list.
  • clone() -> list: Returns a copy of the list.
  • countOf(value: dynamic) -> int: Returns the number of occurrences of a value.
  • find(value: dynamic) -> int: Returns the index of the first occurrence of a value.
  • insertAt(index: int, value: dynamic) -> list: Inserts a value at the specified index.
  • pull() -> dynamic: Removes and returns the last element.
  • removeValue(value: dynamic) -> list: Removes the first occurrence of a value.
  • order() -> list: Sorts the list in place.
  • merge(other: list) -> list: Merges another list into the current list.

Hash Methods

  • keys() -> list: Returns a list of keys from a hash.
  • values() -> list: Returns a list of values from a hash.
  • length() -> int: Returns the number of key-value pairs in a hash.
  • wipe() to clear a hash completely
  • clone() when you need a copy to avoid modifying the original
  • take() when you need to remove and use a specific key-value pair
  • take_last() when you need to remove and use the last key-value pair
  • ensure() to safely get or set a default value for a key
  • merge() to combine hashes, being aware that it overwrites existing keys
// List method examples
numbers: list = [1, 2, 3];
numbers.push(4);  // [1, 2, 3, 4]
numbers.order();  // [1, 2, 3, 4]
numbers.insertAt(1, 5);  // [1, 5, 2, 3, 4]
numbers.removeValue(2);  // [1, 5, 3, 4]
numbers.pull();  // Returns 4, numbers is [1, 5, 3]
numbers.empty();  // []

list1: list = [1, 2];
list2: list = [3, 4];
list1.merge(list2);  // [1, 2, 3, 4]

// Hash method examples
user: hash = {"name": "Alice", "age": 25};
keys = user.keys();  // ["name", "age"]
values = user.values();  // ["Alice", 25]
pairs = user.pairs();  // [["name", "Alice"], ["age", 25]]

user.wipe();  // {}
user.ensure("score", 0);  // {"score": 0}

user1: hash = {"name": "Bob"};
user2: hash = {"age": 30};
user1.merge(user2);  // {"name": "Bob", "age": 30}

pair = user1.take("name");  // ["name", "Bob"], user1 is {"age": 30}
last = user1.take_last();  // ["age", 30], user1 is {}

copy = user2.clone();  // {"age": 30}
count = user2.length();  // 1

Variable Scoping

Echo provides explicit control over variable scoping and mutability through the use and use mut statements.

// Use a single variable
use counter;

// Use multiple variables
use name, score;

// Use mut for a single variable
use mut counter;

// Use mut for multiple variables
use mut name, score;

// Global variables
counter: int = 0;
name: str = "Global";

// Function with immutable access
fn readOnly() -> void {
    use counter;  // Read-only access
    say("Counter value:", counter);
}

// Function with mutable access
fn modifyCounter() -> void {
    use mut counter;  // Mutable access
    counter = counter + 1;
}
Best Practices
  • Always use use to make dependencies explicit
  • Use use mut only when you need to modify the variable
  • Be mindful of variable shadowing in nested functions
  • Consider using immutable imports by default for better code safety

Variable Watching

Track variable changes during program execution with the watch statement.

// Watch a single variable
watch counter;

// Watch multiple variables
watch x, y, z;

// Example with variable watching
counter: int = 10;
watch counter;
counter = 20;  // Output: WATCH: counter changed to 20 (in global)
counter = 30;  // Output: WATCH: counter changed to 30 (in global)
Features
  • Watch multiple variables in a single statement
  • Track changes in both global and local scopes
  • Monitor modifications through assignments and method calls
  • Automatically unwatch variables when they go out of scope

Known Issues

Method Name Conflicts

Currently, variables cannot be named the same as built-in methods. For example:

// These will cause syntax errors:
keys: list = user.keys();
values: list = user.values();
upperCase: int = 10;

This is a temporary limitation that will be fixed in a future update. Most programming languages allow using method names as variable names, and Echo will follow this standard behavior.

Error Messages

Current error messages are basic and may not provide enough context for debugging. We are working on:

  • More descriptive error messages
  • Better line and column information
  • Custom error types for different categories of errors
  • Improved error handling system

Type System Limitations

Some type system features are still under development:

  • No support for null values yet
  • Limited type inference
  • No generic types
  • No type aliases

String Quote Inconsistency

Currently, there's an inconsistency in how string quotes are handled:

// Double quotes work as expected
say("hello");  // Output: hello

// Single quotes print the quotes themselves
say('hello');  // Output: 'hello'

This is a temporary limitation. In a future update, both single and double quotes will be supported for string literals.

Hash Access Quote Requirements

Hash keys currently only support double quotes, not single quotes:

// This works correctly
say(user["name"]);  // Output: John

// This causes a syntax error
say(user['name']);  // Syntax Error
say("User ${user["name"]} (age ${user["age"]})");  // Syntax Error
say("Average score: ${user["scores"]["math"]}");  // Syntax Error

This limitation will be addressed in a future update to support both single and double quotes for hash key access.
