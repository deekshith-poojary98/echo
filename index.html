Introduction

Neura is a statically-typed, Python-based programming language with a clean and expressive syntax. It draws inspiration from C-style structure, modern scripting languages, and Pythonic readability. Designed to be beginner-friendly yet powerful, Neura supports strong type annotations, readable control flow, and simplified syntax for rapid development. 

fn greet(name: str) -> void {
    say("Hello, ", name);
}

greet("Neura");

x: int = 10;
y: float = 20.5;
name: str = "Alice";
is_valid: bool = true;

if x > 5 {
    say("x is greater than 5");
} else {
    say("x is 5 or less");
}

for i in 0..5 {
    say("Loop value:", i);
}

This example demonstrates Neura's basic syntax including function declarations, conditionals, loops, and type-safe variables.

Variables

Variables in Neura are strongly typed and must be declared with their type before use. The syntax follows the pattern variable_name: type = value.

Variable Declaration

Here's a comprehensive example of declaring different types of variables:

// Basic types
x: int = 10;            // Integer variable
y: float = 100.12;      // Float variable
name: str = "John";    // String variable
is_ready: bool = true;    // Boolean variable
anything: dynamic = 10;   // Dynamic variable

// Collections
numbers: list = [1, 2, 3];  // List of integers
mixed: list = [1, "two", 3.0];  // List with mixed types
user: hash = {
    "name": "Alice",
    "age": 25,
    "active": true
};  // Hash (HashMap) with string keys

Variable Reassignment

Variables can be reassigned, but the new value must match the declared type:

x = 20;  // Valid: same type (int)
name = "Bob";  // Valid: same type (str)
// x = "hello";  // Error: cannot assign string to int

// Dynamic variables can change type
anything = "Now I'm a string!";
anything = 42;
anything = true;
Important Notes
  • All variables must be declared with a type
  • Variables cannot change their type after declaration (except dynamic)
  • Variables have block scope and are not accessible outside their block (WIP)
  • Variable names must start with a letter or underscore and can contain letters, numbers, and underscores
  • Variable names are case-sensitive
Best Practices
  • Use descriptive variable names that indicate their purpose
  • Declare variables as close as possible to where they are used
  • Use dynamic type sparingly and only when necessary
  • Initialize variables when declaring them when possible
  • Use constants for values that shouldn't change

Data Types

Neura is a statically-typed language that supports several built-in data types. Each type has specific characteristics and use cases.

int

Integer numbers for whole-value arithmetic operations and counting.

  • Range: -2^63 to 2^63-1
  • No decimal points
  • Supports all arithmetic operations
age: int = 25;
count: int = -100;
result: int = age + count;
float

Floating-point numbers for precise decimal calculations.

  • 64-bit double precision
  • Supports scientific notation
  • Handles decimal arithmetic
pi: float = 3.14159;
scientific: float = 1.23e-4;
result: float = pi * 2.0;
str

String data for text manipulation and storage.

  • UTF-8 encoded
  • Immutable
  • Supports string interpolation
name: str = "Alice";
message: str = "Hello, " + name;
multiline: str = "Line 1\nLine 2";
bool

Boolean values for logical operations.

  • Values: true or false
  • Used in conditionals
  • Result of comparisons
is_valid: bool = true;
is_empty: bool = false;
result: bool = is_valid && !is_empty;
list

Ordered collections of elements.

  • Zero-based indexing
  • Mutable
  • Can contain mixed types
numbers: list = [1, 2, 3];
mixed: list = [1, "two", 3.0];
hash

Key-value mappings for structured data.

  • String keys only
  • Mutable
  • Supports nested structures
user: hash = {
    "name": "Alice",
    "age": 25
};
user["age"] = 26;  // Update value
dynamic

Flexible type for runtime type changes.

  • Can hold any type
  • Type checked at runtime
  • Use with caution
value: dynamic = 10;
value = "hello";
value = true;
Type Safety
  • All variables must be explicitly typed
  • Type checking is performed at compile time
  • Type mismatches result in compilation errors
  • Use dynamic only when necessary
  • Type annotations are mandatory for function parameters and return types

Type Conversion

Neura provides built-in functions for type conversion:

  • asInt(): Convert to integer
  • asFloat(): Convert to float
  • asString(): Convert to string
  • asBool(): Convert to boolean
// String to integer conversion
num_str: str = "123";
num: int = asInt(num_str);

// Float to string conversion
price: float = 19.99;
price_str: str = asString(price);

// String to float conversion
temp_str: str = "98.6";
temperature: float = asFloat(temp_str);

// Integer to boolean conversion
value: int = 1;
flag: bool = asBool(value);  // true

// String to boolean conversion
str_true: str = "true";
bool_value: bool = asBool(str_true);

Control Flow

Neura provides comprehensive control flow structures to manage program execution. These include conditional statements, loops, and control transfer statements.

Conditional Statements

Use if, else if, and else statements to control program flow based on conditions. All conditions must evaluate to a boolean value.

if user_age > 18 {
    say("User is an adult.");
} else if user_age == 18 {
    say("User is exactly 18 years old.");
} else {
    say("User is a minor.");
}

// Nested conditionals
if is_logged_in {
    if has_permission {
        say("Access granted.");
    } else {
        say("Insufficient permissions.");
    }
} else {
    say("Please log in first.");
}

While Loop

The while loop executes a block of code repeatedly as long as a condition is true. Use with caution to avoid infinite loops.

// Basic while loop
count: int = 0;
while count < 5 {
    say("Count:", count);
    count = count + 1;
}

// While loop with break
found: bool = false;
index: int = 0;
while !found && index < items.length() {
    if items[index] == target {
        found = true;
        break;
    }
    index = index + 1;
}

For Loop

The for loop iterates through a range of values with optional step increment using the by keyword. Use .. for inclusive ranges (includes end value) and ... for exclusive ranges (excludes end value). The break statement exits the loop early, while continue skips to the next iteration.

// Basic for loop (exclusive range)
for i in 0...10 {
    say("Iteration:", i);  // Prints 0 to 9
}

// For loop with step (inclusive range)
for i in 0..10 by 2 {
    say("Even iteration:", i);  // Prints 0,2,4,6,8,10
}

// Reverse for loop (inclusive range)
for i in 10..0 by -1 {
    say("Countdown:", i);  // Prints 10 down to 0
}

// Loop with break statement
for i in 1...100 {
    if i > 5 {
        break;  // Exit loop when i > 5
    }
    say("Number:", i);  // Prints 1 to 5
}

// Loop with continue statement
for i in 1...5 {
    if i % 2 == 0 {
        continue;  // Skip even numbers
    }
    say("Odd number:", i);  // Prints 1, 3
}

Foreach Loop

The foreach loop iterates through all elements in a collection such as a list or hash. 

// Foreach with list
fruits: list = ["apple", "banana", "cherry"];
foreach fruit in fruits {
    say("Fruit:", fruit);
}

// Foreach with string
text: str = "Hello, World!";
foreach char in text {
    say("Character:", char);
}

// Foreach with hash
user: hash = {
    "name": "Alice",
    "age": 25,
    "city": "New York"
};
foreach key in user {
    say(key, ":", user[key]);
}
Control Flow Tips
  • Always use curly braces {} for control structures
  • Be careful with infinite loops in while loops
  • Use break and continue sparingly
  • Consider using foreach instead of indexed loops when possible
  • Keep loop bodies small and focused
Best Practices
  • Use meaningful variable names in loops
  • Keep conditions simple and readable
  • Use early returns in functions to reduce nesting
  • Consider using guard clauses for error conditions
  • Document complex control flow with comments

Functions

Functions in Neura are defined using the fn keyword, with mandatory type annotations for parameters and return types. The syntax is:

fn function_name(param1: type1, param2: type2) -> return_type { ... }

Return Types

Every function must specify a return type using the -> operator. Neura supports the following return types:

  • void - For functions that don't return a value
  • int - For functions returning integer values
  • float - For functions returning floating-point numbers
  • str - For functions returning strings
  • bool - For functions returning boolean values
  • list - For functions returning lists
  • hash - For functions returning hash maps
  • dynamic - For functions that may return different types
Important Notes
  • All functions have strict type checking for their parameters
  • Type conversion functions will raise errors if conversion is not possible
  • String functions only work on string values and will raise TypeError otherwise
  • Collection functions have specific type requirements (e.g., length() works on strings, lists, and hashes)
  • The wait() function is useful for creating delays in animations or timed operations

Type Safety

Neura enforces strict type checking for function returns. If a function declares a specific return type, it must return a value of that type, or a TypeError will be raised. Here are some examples: 

// Function with void return type - doesn't return a value
fn greet(name: str) -> void {
    say("Hello, ", name);
}

// Function that returns an integer
fn calculate_sum(a: int, b: int) -> int {
    return a + b;  // Returns int as specified
}

// Function that returns a string
fn get_full_name(first: str, last: str) -> str {
    return first + " " + last;
}

// Function with dynamic return type
fn get_value(key: str) -> dynamic {
    if key == "age" {
        return 25;  // Returns int
    } else {
        return "unknown";  // Returns string
    }
}

Type Errors

The following examples demonstrate common type errors that Neura will catch:

// Error: Function declares int return type but returns float
fn calculate_average(a: int, b: int) -> int {
    return (a + b) / 2.0;  // TypeError: Expected int, got float
}

// Error: Function declares void return type but returns a value
fn print_message(msg: str) -> void {
    say(msg);
    return "done";  // TypeError: void function cannot return a value
}

// Error: Missing return type annotation
fn add_numbers(a: int, b: int) {  // SyntaxError: Return type annotation required
    return a + b;
}
Best Practices
  • Always specify return types for clarity and type safety
  • Use void for functions that perform actions but don't return values
  • Use dynamic sparingly, only when a function truly needs to return different types
  • Match the declared return type exactly - type coercion is not automatic
  • Consider breaking down complex functions that might return different types into smaller, type-specific functions
Important Notes
  • The void type can only be used as a return type, not as a variable type
  • All functions must have an explicit return type annotation
  • Type checking is performed at runtime to ensure type safety
  • Return type mismatches will raise a TypeError

Built-in Functions

Neura provides a set of built-in functions for common operations. These functions are categorized by their purpose and include detailed type information.

Input/Output Functions
  • ask(prompt: str) -> str: Prompts the user for input with the given message and returns the user's input as a string.
  • say(...values: dynamic) -> void: Prints values to the console, accepting multiple arguments of any type.
  • wait(seconds: float) -> void: Pauses execution for the specified number of seconds.
// Input and output
name = ask("What's your name? ");
say("Hello, ", name);
wait(1.5);  // Wait for 1.5 seconds
Type Conversion
  • asInt(value: dynamic) -> int: Converts a value to an integer, raising an error if conversion is not possible.
  • asFloat(value: dynamic) -> float: Converts a value to a float, raising an error if conversion is not possible.
  • asBool(value: dynamic) -> bool: Converts a value to a boolean (true/false).
  • asString(value: dynamic) -> str: Converts any value to its string representation.
// Type conversion
age = asInt(ask("How old are you? "));
price = asFloat("19.99");
is_valid = asBool("true");
name = asString(age);
Type Information
  • type() -> str: Returns the type of a value as a string (e.g., "int", "str", "list", "hash").
// Type information
value: int = 10;
say(value.type());  // "int"

value: str = "Hello";
say(value.type());  // "str"
String Functions
  • trim(text: str) -> str: Removes whitespace from both ends of a string.
  • upperCase(text: str) -> str: Converts all characters in a string to uppercase.
  • lowerCase(text: str) -> str: Converts all characters in a string to lowercase.
// String manipulation
text: str = "  Hello World  ";
clean: str = trim(text);  // "Hello World"
upper: str = upperCase(clean);  // "HELLO WORLD"
lower: str = lowerCase(clean);  // "hello world"
Collection Functions
  • length() -> int: Returns the length of a string, list, or hash.
  • reverse() -> list | str: Reverses the elements of a list or string.
  • keys() -> list: Returns a list of keys from a hash.
  • values() -> list: Returns a list of values from a hash.
// Collection operations
user: hash = {"name": "Alice", "age": 25};
keys = keys(user);  // ["name", "age"]
values = values(user);  // ["Alice", 25]

// Length and reverse examples
text: str = "Hello";
say(text.length());  // 5
say(text.reverse());  // "olleH"

nums: list = [1, 2, 3];
say(nums.length());  // 3
say(nums.reverse());  // [3, 2, 1]
List Methods
  • push(value: dynamic) -> list: Adds an element to the end of the list and returns the modified list.
  • empty() -> list: Removes all elements from the list and returns the empty list.
  • clone() -> list: Returns a copy of the list.
  • countOf(value: dynamic) -> int: Returns the number of occurrences of a value in the list.
  • find(value: dynamic) -> int: Returns the index of the first occurrence of a value, or raises an error if not found.
  • insertAt(index: int, value: dynamic) -> list: Inserts a value at the specified index and returns the modified list.
  • pull() -> dynamic: Removes and returns the element at the specified index (or the last element if no index is provided).
  • removeValue(value: dynamic) -> list: Removes the first occurrence of a value and returns the modified list.
  • order() -> list: Sorts the list in place and returns the sorted list.
  • merge(other: list) -> list: Merges another list into the current list and returns the modified list.

List Methods Examples

// Basic list operations
numbers: list = [1, 2, 3];
say(numbers.length());  // 3

// Adding elements
numbers.push(4);
say(numbers);  // [1, 2, 3, 4]

// Counting occurrences
numbers.push(2);
say(numbers.countOf(2));  // 2

// Finding elements
say(numbers.find(3));  // 2

// Inserting elements
numbers.insertAt(1, 10);
say(numbers);  // [1, 10, 2, 3, 4, 2]

// Removing elements
removed = numbers.pull(2);
say(removed);  // 2
say(numbers);  // [1, 10, 3, 4, 2]

// Removing values
numbers.removeValue(2);
say(numbers);  // [1, 10, 3, 4]

// Sorting
numbers.order();
say(numbers);  // [1, 3, 4, 10]

// Merging lists
other: list = [20, 30];
numbers.merge(other);
say(numbers);  // [1, 3, 4, 10, 20, 30]

// Cloning lists
copy = numbers.clone();
say(copy);  // [1, 3, 4, 10, 20, 30]

// Emptying lists
numbers.empty();
say(numbers);  // []
